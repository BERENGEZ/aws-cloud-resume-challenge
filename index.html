🌐

1. Cloudflare Tunnel

My ISP uses Carrier-Grade NAT (CGNAT), which prevents inbound connections. Cloudflare Tunnel provides secure public access without exposing router ports or requiring port forwarding.

🖥️

2. Proxmox & Ubuntu

Proxmox hosts an Ubuntu Server virtual machine with dedicated CPU and memory resources to isolate the application stack from the host OS.

🚦

3. Nginx Web Server

Acts as a reverse proxy. It serves the static frontend assets and forwards /api/ requests to the backend while handling TLS termination.

⚙️

4. Python Flask & Gunicorn

A Flask REST API handles application requests and serves telemetry data. Gunicorn and systemd manage the service in production to ensure high availability and zero manual restarts.

🗄️

5. SQLite Database

Stores visitor metrics locally. The database is accessible only through the internal Flask API and is never exposed directly to the internet.

🚀

6. CI/CD Pipeline

GitHub Actions triggers a self-hosted runner on the Ubuntu VM, which pulls the latest commit and deploys the application automatically upon push.

Challenges Solved & Lessons Learned

  • Network Constraints: Overcame ISP CGNAT limitations by implementing Cloudflare Tunnels for secure ingress.
  • Process Supervision: Configured systemd service files to manage Gunicorn, ensuring the API automatically recovers from crashes and reboots.
  • Deployment Automation: Replaced manual SFTP file transfers with a self-hosted GitHub Actions runner for instant, reliable CI/CD.
  • Reverse Proxying: Configured Nginx routing blocks to separate static file delivery from dynamic API request forwarding.
